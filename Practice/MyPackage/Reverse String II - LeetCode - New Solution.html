Intuition

The problem repeats this rule:
For every block of 2k characters
Reverse the first k, keep the next k unchanged.

So the string is processed in chunks of size 2k

1.Reverse the first k
2.Keep the next k as it is
3.If remaining characters < k reverse all
4.If remaining between k and 2k reverse first k, keep rest

Approach

1.Convert string to character array
Because strings are immutable in Java.

2.Iterate in steps of 2k
for ( i = 0; i < n; i += 2 * k)

3.Define reversal range
Start = i
End = min(i + k - 1, n - 1)
(Handles edge cases safely)

4.Reverse characters between start and end
->Use two-pointer swap (left++, right--)

5.Continue until string ends
6.Convert array back to string and return

Complexity

  • Time complexity:O(n)
    Space complexity:O(n) due to char arr[];

Code

Java
class Solution {
    public String reverseStr(String s, int k) {
        char [] arr = s.toCharArray();
       int n=arr.length;
       for(int i=0;i<n;i+=2*k) {
            int left=i;
            int right=Math.min(i+k-1,n-1);

            while(left < right) {
                char temp = arr[left];
                arr[left] = arr[right];
                arr[right] = temp;
                left++;
                right--;
            }
       }
       return new String(arr);   
    }
}
